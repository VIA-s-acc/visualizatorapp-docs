Overview

Desktop application

  • Load a folder with sensor CSV files
  • Select one or more channels per sensor
  • Render time-domain signals
  • Compute and render frequency spectra (FFT baseline)
  • Reset chart views and inspect multiple channels in parallel

Processing core

  • CSV loading from:
    • header row in each CSV
    • optional channels.txt
  • Segmentation methods:
    • fixed-size windows
    • energy-based segmentation
    • variance-based segmentation
    • change-point detection (including PELT fallback path)
    • cross-correlation gating
  • Spectral transforms:
    • FFT
    • Welch
    • Multitaper (DPSS)
    • CWT
    • STFT
  • Spectrum metrics:
    • absolute range metrics (area, peak, centroid, RMS, flatness, rolloff, etc.)
    • pairwise range comparisons (ratio, delta, correlation, KL, cosine)

Synthetic data generation

  • Multi-sensor synthetic dataset generation via CLI
  • Scenarios:
    • stationary
    • events
    • regimes
    • chirp
    • mixed
    • harmonic
    • correlation
  • Optional validation and batch generation
  • Ground-truth metadata in _meta.json

Synthetic Data Generator (CLI)

Quick start

generate-sensor-data --scenario mixed --output ./generated_data

Or:

python generate_sensor_data.py --scenario mixed --output ./generated_data

CLI options

Option Short Type Default Description
--output -o str ./generated_data Output folder
--sensors -s str D1,D2,D3 Comma-separated sensor names
--channels -c str x,y,z Comma-separated channel names
--duration -d float 10.0 Signal duration in seconds
--sample-rate -r int 50 Sampling rate in Hz
--scenario str mixed Generation scenario
--header-mode str csv Header mode: csv or txt
--noise -n float 0.05 Base noise level
--seed int None Random seed
--config str None JSON config path
--include-time / --no-include-time bool True Include t column
--num-events int 3 Event count for event-based scenarios
--batch int 1 Number of generated runs
--validate flag False Run structural/FFT validation
--verbose -v flag False Verbose console output

Example commands

# Basic mixed dataset
generate-sensor-data --scenario mixed --output ./generated_data
# Reproducible event dataset with validation
generate-sensor-data --scenario events --num-events 5 --seed 42 --validate -v
# Header mapping through channels.txt
generate-sensor-data --header-mode txt --output ./generated_data_txt
# Batch generation
generate-sensor-data --scenario chirp --batch 10 --output ./batch_data
# JSON config-driven generation
generate-sensor-data --config ./scenarios/vibration_test.json --output ./run_cfg

Input Data Format

Supported input formats:

  1. Header in CSV first row:
t,x,y,z
0.00,0.2,0.4,0.1
0.02,0.3,0.1,0.6
  1. Header mapping in channels.txt, CSV files without header rows:
D1:t,x,y,z
D2:t,x,y,z

If no time column is present (t, time, timestamp), the app builds a synthetic time axis from sample index and configured sample rate.

Troubleshooting

<tt>ModuleNotFoundError</tt> during test collection

  • Install project dependencies in the active environment: pip install -e .[dev]
  • Run tests from the project root (Signals&SpectrumViewer_V1)

GUI does not start

  • Verify PyQt6 is installed in the same environment used for launch
  • Confirm active interpreter/environment is correct

Different results across environments

  • Set a fixed --seed
  • Verify environment consistency (conda info --envs, where python)
